# OpenSpec

## Role

External workflow solution for spec-driven development. Treat it as a framework
for turning intent into structured specifications, tasks, and implementation
flow.

## Repository

- Upstream: https://github.com/Fission-AI/OpenSpec
- Import strategy: `index-only`
- Checked commit: `11b269061897b011075a984c6c95d970a5533a66`
- Last checked: 2026-05-28

## Local Policy

Do not vendor the full repository here. Keep local adapter notes and examples
only when they describe how this repository wants to use OpenSpec.

Do not split OpenSpec into standalone skills unless a specific command, prompt,
or workflow step is customized and useful outside the OpenSpec framework.

## When To Use

- Requirements clarification.
- Spec-driven development.
- Task breakdown before implementation.
- Consistent handoff between planning and coding.

## Boundary

OpenSpec owns the end-to-end spec lifecycle. This repository owns only the
index, local usage decision, and any small adapter notes needed to use it with
personal workflows.
